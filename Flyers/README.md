# 自由研究プロジェクト：アジャイル開発手法を用いたゲーム開発およびプロセス学習の報告

## 1. プロジェクトの背景および目的
本プロジェクトは、デジタルコンテンツの制作プロセスを通じて、現代社会において不可欠とされる「アジャイル型問題解決能力」および「ICT活用スキル」の習得を主眼としています。
単なる完成品（ゲーム）の作成というアウトプットに留まらず、開発過程における「試行錯誤」や「機能改善の論理的経緯」そのものを学術的な成果物と定義し、プログラミング教育の新たな実践的モデルを提示することを目的とします。

## 2. 重点コンセプトの解説

### ① スクラム思考の組織的習得
本活動では、現代の開発現場で標準的に用いられる「スクラム（Scrum）」の概念を導入しています。
*   **イテレーション（反復）開発:** 短期間のサイクル（スプリント）で設計・実装・検証を繰り返し、成果物を段階的にブラッシュアップする手法を実践します。
*   **バックログ管理と優先順位付け:** 利用者（子供たち）の潜在的な要望を「機能要件」として抽出・構造化し、限られたリソースの中でどの機能を優先して実装すべきかという意思決定プロセスを体験させます。

### ② 実戦的エンジニアリングスキルの習得
理論上の学習のみならず、プロフェッショナルが使用する技術環境を実体験することで、高い実効性を備えた技術習得を目指します。
*   **概念の具現化:** 抽象的なアイデアを技術的要件へと変換する「エンジニアリング・シンキング」を育成します。
*   **標準的開発環境の活用:** VS Code等の統合開発環境、GitHubによるバージョン管理、およびGitHub Pagesを用いたデプロイメントといった一連のワークフローを経験し、実務レベルの開発基準に触れることで、技術に対する深い洞察力を養います。

### ③ 成果物の公開と持続性
作成されたゲームは、ブラウザを通じてインターネット上に公開され、世界中の誰からもアクセス可能な環境に置かれます。これにより、作成した成果物が社会に対して物理的な価値を持つことの理解を深めるとともに、提出後も継続的な改善が可能な「動的なソフトウェア資産」としての運用を経験させます。

### ④ プロセス重視の研究成果物
本研究の最大の特徴は、成果物の「完成度」だけでなく、その「開発履歴（プロセス）」を重視している点です。
*   **開発ログの構築:** 実装に至るまでの仮説検証や、予期せぬエラーに対するデバッグの履歴を開発ログとして記録・蓄積します。
*   **AIを用いた情報の構造化:** 生成AIを単なる回答生成ツールではなく、自己の思考の整理・論理的構成を行う「知的支援ツール」として活用し、開発プロセスの言語化・体系化を図ることで、客観的な研究レポートの作成を実現します。

## 3. 指導体制：役割分担による教育的アプローチ
指導者と学習者の間には明確な責任の境界線を設けております。
*   **学習者の役割:** プロダクトオーナーとして、ゲームの仕様策定および機能改善の意思決定を行う。
*   **指導者の役割:** 実装エンジニアとして、技術的担保および品質管理を行う。

この役割分担により、学習者は自身の想像力を技術によって具現化する「自己効力感」を得ると同時に、プロフェッショナルが有する高い技術力への敬意と明確な学習目標を醸成することを目指しています。
